### Full-arc flake fixed (#114, closes #107)

Root cause: the game reads one-shot inputs with Phaser `JustDown`, which only
sees a key that is down on a frame boundary. An instantaneous `press` can land
entirely between two starved frames on a loaded runner and be lost. Driving
survived (arrow keys are held across frames); every one-shot press was fragile.
Dominant symptom: a dropped skill-panel close left the panel open, and the
contract-accept handler ignores number keys while it is open, so every later
`1` was swallowed and the arc stalled with an opaque `blockade_broken` failure.

Fix (test harness only, no game code change): `setSkillPanel` holds `k` and
polls for the toggle; new `tapKey` holds any one-shot key across a frame and all
arc one-shot presses route through it; a no-progress watchdog fails loudly with
live state (its signature excludes `atHome` so the wagon coasting on/off the
home tile cannot mask a stall); a `test:arc` script pins `nice -n 19
--workers=1` for safe local runs. Validated full-arc 3/3 under load; on-merge
Arc green on `main`.
