dialogue engine, dialogue wiring with the first conversation, spoke

## Completed

Narrative spine (owner decisions captured via prompt):
- `docs/design/04_storyline.md` (#41): the Blockade. The borderland's silence is
  engineered by a rival information network; the courier is the thread that
  reconnects it. Hub sets up, spokes reveal one half each (either order), hub
  resolves; region resolves, larger world left open.

M5 build (all squash-merged to main, each green behind CI, deployed):
- #42 `src/systems/dialogue.ts`: pure branching dialogue engine with story
  flags (allOf/noneOf gating, immutable flag application, validation).
- #43 `src/scenes/map-hud.ts`: extracted the HUD/overlay layer out of
  `map-scene.ts` (1608 to 1363 lines) so the dialogue UI had a home. MapHud owns
  all HUD GameObjects, the minimap renderer, toasts, STATUS_COLOR + HUD style.
- #44: dialogue wired into the game. Talk with E at a settlement, number keys
  choose, Esc closes, movement is modal/frozen. Story flags persisted as an
  optional `storyFlags` save field (no version bump). First conversation: the
  Greywater postmaster (Act 1). The scene hands the engine persisted flags plus
  flags derived from live world-state (e.g. `home_reconnected` when the region
  is cleared), so a choice can gate on a real fact without storing it.
- #45: spoke conversations. Tidewatch harbormaster (Saltreach, the method:
  vanished courier Wrenn, the birds at Cormorant Rock) and Mossgate warden
  (Fenmarch, the cost). Each reveal gated on its region reconnected. Also now
  allocate all six number keys (was sized to contract count).
- #46: social skills hook dialogue. A skill grants a derived `skill_<id>` flag
  while owned (`skillFlag`/`derivedSkillFlags`), folded into the scene's
  effective flags. New Cipher skill (reading carried secrets) unlocks a
  postmaster line.
- #47: missions (M5.3). Pure `src/systems/mission-system.ts` (steps derived from
  completed contracts + story flags + visits; no persisted mission state).
  `src/data/missions.ts`: Greybridge spine, two spoke chains (gated on the hub
  reveal), and a Greywater capstone (gated on both spoke reveals, sets
  `blockade_broken`). Active mission step leads the idle HUD objective; journal
  has a Story section.

## Current state

What works:
- `main` green (check + e2e) and deployed. Working tree clean.
- 493 unit tests + 13 Playwright e2e. New e2e: dialogue, social-skill, missions.
- The full arc is playable: talk to the postmaster, reconnect Greybridge, get
  the reveal, do the spokes (either order), return for the capstone.
- Milestones doc M5.2 and M5.3 are ticked. ADR 0004 and the storyline doc are
  the source of truth for the narrative shape.

What is incomplete (future scope, deliberately not started):
- Non-combat road encounters (ADR 0004 section 6).
- Deeper world-state consequences: contracts that appear or withdraw as the arc
  advances, price/reward shifts (M5.4 "still to do").
- Content beyond the three current regions (who commands the Blockade, whether
  the Fenmarch dark is real) - left open by the resolution decision.

Known issues / smells:
- Flaky e2e: `travel.spec.ts:61` (Fenmarch travel, a single T keypress) has
  flaked across several PRs under CI load. #47 raised the travelTo timeout to
  30s, which should help; re-running the failed job has always gone green. If it
  keeps flaking, make the T-press re-pressing more aggressive or assert arrival
  more tolerantly.
- `map-scene.ts` is back around 1500 lines after the dialogue and mission
  wiring. Still far better than before the MapHud split, but input handling
  (handle*Input) or the dialogue controller could be extracted next if it grows.
