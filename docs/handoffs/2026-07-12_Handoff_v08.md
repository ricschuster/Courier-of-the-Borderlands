### PR #137 merged (branch `fix/arc-coast-past-home`)

The fix, top to bottom, was already partly built in v07 (`?nowear` flag,
frame-anchored harness). This session added the review findings and shipped it:

- **`?nowear` e2e flag** (from v07): the Arc is a reachability / soft-lock
  guard, not a travel-sink test, so wear is disabled on its critical path.
  Removed the limp-crawl failure class outright.
- **Frame-anchored harness** (from v07): waits on the game's own frame counter,
  not wall-clock, so the step budget no longer depends on runner speed.
- **Review findings applied this session** (adversarial review of the diff):
  1. `waitForFrames` now anchors on the hook's first visible frame instead of a
     `?? -1` fallback that completed instantly mid scene-restart (voiding the
     N-frame hold guarantee).
  2. seat radius widened 2 -> 3 tiles (`e2eSeat` in `map-scene.ts`), so a
     worst-case full-kit coast settles instead of hard-failing the re-seat home.
  3. `driveToTile` samples every frame, not every two: held arrow keys read with
     `isDown`, so one frame both observes them and moves, and one frame moves
     under a tile, so the goal tile can no longer be hopped (a false "stuck").
- **Temporary `fix/arc-coast-past-home` push trigger removed** from `arc.yml`.
  `workflow_dispatch` kept (run the Arc manually on any branch).
