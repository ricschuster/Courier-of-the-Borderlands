## Completed

Camera follow + minimap (PR #27):
- `setupCamera()` in `map-scene.ts`: binds the main camera to the map and calls
  `startFollow` on the courier only when the map exceeds the 960x540 viewport.
  Maps that fit stay static and centred (bounds set with `centerOn`, so Phaser
  leaves a sub-viewport map put; follow clamps at the edges so no background
  bleeds in). Smoothed by a new `CAMERA_LERP` config value (0.12).
- The map now draws at the world origin (`mapOriginY = 0`); the camera handles
  centring. Tile coordinates are unchanged (origin and spawn shift together),
  so e2e tile assertions and saved fog indices are unaffected.
- Every HUD element (title, stats, hint, board, journal, summary, legend,
  minimap, toasts) pinned with `setScrollFactor(0)` so it no longer scrolls
  with the world.
- Minimap cell size shrinks to fit a bounded box (`MINIMAP_MAX_PX = 192`) on
  large maps; small maps keep the current 6px/tile look.
- Verified both camera branches with screenshots on a throwaway 30x22 region
  (small map centred + pinned HUD; large map clamps at the corner and follows
  to centre), then removed the scaffolding.

Seeded PRNG (PR #28):
- `src/systems/rng.ts`: pure deterministic mulberry32 generator with `next()`
  (float [0,1)), `nextInt(maxExclusive)`, and `pick(items)`. Same seed
  reproduces the same sequence.
- `pickWeather(rng)` in `weather.ts` replaces the scene's
  `Math.floor(Math.random() * WEATHERS.length)`; the scene seeds from
  `Date.now()`, so weather still varies per run but the roll is deterministic
  and testable. This removes the last `Math.random()` from `src/`.

Region invariant tests (PR #29):
- `tests/unit/region-invariants.test.ts` runs the game's own 4-directional BFS
  over every authored region, so a pass means the route exists in play. Per
  region: spawn passable; every settlement and gateway reachable from home with
  no unlocks; every contract has real endpoints with an existing pickup ->
  destination route. Per ford region: ford tiles keyed to the unlock id,
  blocked at base and passable once unlocked, signpost on passable ground; and
  unlocking the ford opens a crossing strictly shorter than its detour.
- The ford-shortcut check compares bank-to-bank distance locked vs unlocked
  across the whole ford group, so multi-tile fords work. It has teeth: it
  failed for the Greybridge and Fenmarch two-tile fords until the bank-pairing
  spanned the full crossing.
