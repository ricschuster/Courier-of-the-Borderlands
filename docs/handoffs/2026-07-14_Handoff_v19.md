## Current state

- **What works:** `main` is green and released at **v0.2.0**. Live surfaces:
  the game, `telemetry.html`, and `manual.html`. Preview deploys work per PR.
  release-please opens and updates the release PR automatically.
- **Known gotcha (tracked in #238):** release-please opens its release PR with
  the default `GITHUB_TOKEN`, which does not trigger `on: pull_request` CI, so
  the required checks never run and the PR is BLOCKED. Worked around for 0.2.0
  by close/reopening the PR as a user to trigger CI. The durable fix needs an
  owner-provided PAT/App-token secret (`RELEASE_PLEASE_TOKEN`); the workflow can
  be wired the moment it exists.
- **One-time cosmetic:** the 0.2.0 changelog is comprehensive (whole history)
  rather than incremental, because there was no real `v0.1.0` tag to diff
  against. Now that `v0.2.0` is tagged, future releases will be incremental.
