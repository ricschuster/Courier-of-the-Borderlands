# Session Handoff: 2026-07-06

## Summary

A reflective checkpoint, then three shipped PRs that turn the playtest's
findings into working systems. The session paused first to reassess the whole
approach (continue vs restart); the decision was to keep going on this game and
let a fun signal drive what to build (path A). A 20-minute playtest gave a clean
verdict: the courier fantasy is fine, the missing thing is stakes. Everything
built this session follows from that.

Live game: https://ricschuster.github.io/Courier-of-the-Borderlands/

## Completed

Playtest and direction (no code):
- `docs/design/05_playtest_notes.md`: recorded the session, the disengage moment
  (rewards flatline, so deliveries stop mattering), and the verdict that stakes,
  not the fantasy, are the gap.

PR #37 -- Reconnection slice (first cut of the world-state north star):
- `src/systems/world-state.ts`: per-settlement status (silent / reconnected /
  home) derived purely from delivery history. No new save field, no migration.
- Settlement markers and minimap cells recolour when a delivery reconnects a
  place; the journal became a re-openable status board with a re-readable
  objective and a spine payoff line per reconnected place
  (`src/data/reconnection-notes.ts`).

PR #38 -- Polish and coin sink:
- HUD text now sits on a dark backing so it stays legible over light terrain
  (the help line washed out on discovered tiles).
- Three more Greybridge upgrades raise the coin sink from 150 to 500.
- Gateways that share a tile with a settlement name it in the hint (Southmill).

PR #39 -- Courier experience and skills (M5.1):
- `src/systems/experience.ts`: XP derived from play stats (deliveries, distance,
  discoveries) plus a level curve. Derived, not stored, like world-state.
- `src/systems/skills.ts`: ranked skills (Wayfinder, Teamster, Negotiator) whose
  effects feed the existing pipeline. K panel spends points; HUD shows level.
- Only new save state is chosen skill ranks (optional field, no version bump).

## Current state

What works:
- `main` green (check + e2e) and deployed. Working tree clean.
- 426 unit tests + 10 Playwright e2e. The playthrough e2e now asserts, in a real
  browser, that a delivery reconnects Eastwatch and levels the courier up.
- Three progression axes are now all live and legible: coins to upgrades,
  reputation to trust, experience to skills. World-state makes deliveries matter.

What is incomplete:
- The narrative spine is still undesigned. Reconnection payoff lines exist per
  settlement, but there is no main thread tying the regions into an arc.
- M5.2 (dialogue) and M5.3 (missions) are not started.
- Deferred skills: true "verb" skills needing engine work (terrain access,
  multi-carry) and social skills that depend on dialogue.

Known issues / smells:
- `src/scenes/map-scene.ts` keeps growing (now well over 1400 lines). It is due
  a breakup (HUD, input, overlays into modules) before the dialogue UI lands.
