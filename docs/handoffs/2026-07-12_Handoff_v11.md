## Completed

### Three playtest bugs fixed (merged)

- **Fractional coins (#145).** `repair()` charged `missing * costPerPercent` with
  a float condition, leaking fractional coins into the ledger (a balance read
  957.33). Now charges whole coins matching the quoted `repairCost`.
- **Stale HUD skill points (#146).** XP is continuous (distance + discoveries), so
  a level crossed mid-drive left the wallet showing a stale level/point count
  while the K panel read live. The wallet now refreshes the frame the level
  changes.
- **Arrival banner overlap (#148 -> PR #154).** The persistent arrival toast sat
  over the contract board and the postmaster dialogue. Any conversation now
  clears lingering toasts on open. Verified with before/after screenshots.

### Difficulty locked at a start screen (#150 -> PR #157, merged)

Retired the live in-run G-key difficulty selector for a one-time choice, per the
owner decision below.

- New `TitleScene` difficulty picker (relaxed / standard / demanding), shown only
  on a fresh game. Selecting persists the choice and starts the run.
- `BootScene` now routes: a save in progress resumes straight to the map on its
  locked difficulty; a fresh game opens the picker. Under the e2e hook a fresh
  game skips the picker (`?e2e`), and `?e2e&title=1` forces it so the spec can
  drive it. New game (N) routes back through `BootScene` to re-pick.
- `MapScene`: removed the G key, its live cycling, and the mid-run clamp; the HUD
  shows the locked difficulty read-only. `difficulty.spec` rewritten for the
  picker flow; smoke test updated; ADR 0005 updated. This `TitleScene` is the
  start-screen foundation the rest of the UI pass (#149) builds on.
