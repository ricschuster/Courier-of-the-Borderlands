# Session Handoff: 2026-07-07

## Summary

Built the last open M5.3 item: non-combat road encounters, on top of the
dialogue engine. Encounters fire while driving, open the existing modal
dialogue, and a choice applies a coin or reputation outcome. Pure system, data
content, scene wiring, unit tests, map invariants, and a browser e2e. All checks
green locally.

Live game: https://ricschuster.github.io/Courier-of-the-Borderlands/

## Completed

Road encounters (M5.3 final box, ADR 0004 section 6):
- `src/systems/encounter-system.ts`: pure engine. `RoadEncounter` (id, region,
  trigger tile, a `Dialogue`, and an `outcomes` map). `pickEncounter` selects
  the encounter for the courier's region + tile + flags, skipping any already
  resolved. One-shot is modelled by resolution flags being the keys of
  `outcomes`: once any is set the encounter is spent. `outcomeForFlag`,
  `isEncounterResolved`, `resolutionFlags`, and `validateEncounter` round it out.
  No RNG: encounters are deterministic by location, which keeps them testable.
- `src/data/encounters.ts`: first three encounters, authored as dialogues in the
  Blockade tone. Greybridge stranded courier (7,8): help for +6 coins and +2
  Greywater reputation, or drive on. Saltreach coast-road toll (5,3): pay 15
  coins for +1 Tidewatch standing, or refuse and bull through free. Fenmarch
  washed-out causeway (7,5): tip the warden's boy 2 coins for +1 Mossgate, or
  thank him. Each choice sets a resolution flag; an empty outcome still resolves.
- Scene wiring in `src/scenes/map-scene.ts`: `handleEncounters()` runs in the
  update loop, fires once per tile entered (a `lastEncounterTile` guard stops it
  re-opening while parked on the tile or the instant you step away), skips
  settlement tiles, and routes into the existing `openDialogue`. A resolving
  choice applies its outcome to the ledger before the save write, with a HUD
  toast. `activeEncounter` clears on close. Story flags already persist, so no
  save-format change and no migration.
- Persistence: resolution flags are ordinary `storyFlags`, so a resolved
  encounter stays resolved across reloads with no new save field, matching
  world-state, experience, and missions.

Tests and docs:
- `tests/unit/encounter-system.test.ts`: selection, one-shot resolution,
  outcome lookup, and validation.
- `tests/unit/encounter-invariants.test.ts`: over every authored encounter,
  asserts the trigger tile is base-passable, not on a settlement, in a real
  region, the dialogue is well formed, ids and tiles are unique, and it fires
  when reached and never again once resolved. Mirrors region-invariants.
- `tests/e2e/encounter.spec.ts`: drives onto the Greybridge encounter tile,
  confirms it opens with no key press, takes the help choice, and asserts the
  coin and reputation outcome plus the persisted flag.
- `tests/e2e/drive.ts`: `driveToTile` now dismisses an encounter that opens
  mid-drive (Escape, then continues) so travel through a trigger tile is not
  blocked. Needed because the first encounter sits on the main east road.
- E2E hook `version` bumped 6 -> 7; new `activeEncounterId` field on E2EState.
- Milestones M5.3 box ticked.

## Current state

What works:
- `npm run typecheck`, `npm run lint`, `npm test` (521 unit tests), and
  `npx playwright test` (14 e2e) all green locally. Working tree has the new
  files plus edits to map-scene, drive.ts, and the milestones doc.
- The first three regions each have one road encounter on the natural route.

What is incomplete (future scope):
- Only three encounters, one per region, all on or near the main road. More
  variety and placement (forest ambushes, weather-closed passes) is open.
- Encounter outcomes are coins and reputation only. They do not yet set story
  flags that missions or world-state read, or grant/withdraw contracts. Wiring
  an encounter into the arc (a `requires` gate plus a spine flag) is the natural
  next step and is already supported by the data model.
- The stranded-courier encounter is mandatory on the first eastbound trip (it is
  on the main road). That is by design (roads are gameplay), but if it reads as
  intrusive in a playtest, move it one tile off the through-road.

Known issues / smells:
- `map-scene.ts` grew again with the encounter handlers. Still the same
  input-handling-in-scene pattern flagged in the prior handoff; extracting an
  input or dialogue controller is the eventual cleanup.
- The Fenmarch travel e2e (`travel.spec.ts:61`) remains the historical flake
  under CI load; it passed this session.
