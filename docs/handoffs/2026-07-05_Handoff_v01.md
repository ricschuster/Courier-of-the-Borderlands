## Working on a fresh machine

The previous machine had machine-specific setup that does NOT transfer:

1. Node was installed user-local there. On the new machine, install Node (the
   repo pins the version): `.nvmrc` = 24, `engines.node` >= 22. With nvm:
   `nvm install && nvm use`. Confirm `node -v` and `npm -v`.
2. Clone and install: `git clone <repo>`, `cd`, `npm ci`.
3. For the browser smoke test: `npx playwright install --with-deps chromium`.
4. GitHub CLI auth (needed for the PR/deploy flow): `gh auth login` then
   `gh auth setup-git`.
5. Sanity check: `npm run lint && npm run typecheck && npm test && npm run build`,
   and optionally `npm run test:e2e`.

Note: the assistant's local memory notes (Node setup, deploy approach, GitHub
workflow, quirks) live under `~/.claude/` on the old machine and will not be
present on the new one. This handoff captures the essentials; new memory will
build up again over time.

## GitHub workflow (important, do not skip)

- Do NOT push to `main`. Branch, open a PR, let CI pass, squash-merge,
  auto-delete branch. `main` requires the `check` and `e2e` status checks.
- Deploy is automatic: after CI succeeds on `main`, the Deploy workflow runs
  via `workflow_run` and publishes to the `gh-pages` branch. Do not switch to
  the `actions/deploy-pages` pipeline; it failed opaquely before. Pages is set
  to serve the `gh-pages` branch (legacy build type).
- Dependabot opens weekly npm + actions PRs; merge the safe ones via the PR
  flow. If a PR is BEHIND after another merges, `@dependabot rebase` then merge.
