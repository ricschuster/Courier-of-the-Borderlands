## The playtest result

The signal, in full:

- **Does player-side progression have teeth mid-to-late?** Yes. "Decent
  balance." Capstone on **576 coins**.
- **Does the world feel reactive across a whole run?** Yes.

Why 576 matters: `07_roads_gate_the_wagon.md` was opened against an autonomous
run that reached the capstone on ~2563 idle coins with 0 upgrades fitted and
unspent skill points. It is also the second independent human run to land in the
same place, against ~601c from #109 on 2026-07-12 at the same difficulty. One
run is an anecdote; two runs landing 25 coins apart is the economy holding rather
than drifting.

## Decisions

- **#213 closed as validated rather than tuned.** The issue's own hard rule was
  "reopen a number only with a concrete wrong value from play." "Decent balance"
  is not a wrong value, it is a confirmation. The standing rule survives the
  closure: any future economy change still needs a concrete wrong number from
  play, one lever per round with exact values.
- **#212 closed as met, not built.** It was parked on this playtest to confirm
  the depth was right and to pick which step/contract. The playtest answered the
  question it was gated on in the direction that removes the premise: the world
  already feels reactive, which is the outcome the direction existed to produce.
  Building more world-state depth against an already-working feel is scope
  without a problem behind it. Reopen if a later playtest reports the story
  advancing on bare talk/deliver rather than off world-state.
- **playtest-triage was deliberately not run.** The skill buckets bugs,
  confusions, and complaints into fix-now versus design-call. There were none.
  Running it would have been ceremony over an empty input.

## Risks or decisions needed

- **Shared origin, still unfiled, carried from v21 and v22.** GitHub Pages origin
  is scheme plus host only, so the live game, every PR preview, and
  `telemetry.html` share one `localStorage` bucket including the real save. A
  preview can resume and overwrite a real playthrough. This playtest sidestepped
  it by running on localhost. Still worth filing.
- **The untracked `telemetry-seed.*` files** are still in the repo root and still
  v1 schema, so the dashboard ignores them. Harmless; delete when convenient.
- **#225 (data-schema validation) should be reopened** if data starts loading at
  runtime, or if #229 (visual region editor) lands. A tool that writes data files
  produces content outside the compiler's reach, which is a real module boundary.
