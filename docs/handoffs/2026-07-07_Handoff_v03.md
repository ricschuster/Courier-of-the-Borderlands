# Session Handoff: 2026-07-07

## Summary

Continued the M5 narrative layer with three shipped packages this session: road
encounters (M5.3, #50), a second side-route encounter batch (#51), and the first
M5.4 world-state-consequence slice, arc-driven contract availability (this
handoff, PR pending). Contracts can now be gated on story flags, so reconnecting
a region opens new work.

Live game: https://ricschuster.github.io/Courier-of-the-Borderlands/

## Completed (this package: arc-driven contracts, M5.4)

- `src/systems/contract-system.ts`: `Contract` gained an optional `requires`
  `FlagCondition`. New pure helpers: `isContractAvailable`, `availableContracts`
  (board filter), `contractsInPlay` (progress-count set), and `baseContracts`
  (ungated standing routes). All unit tested.
- `src/scenes/map-scene.ts`: `boardContracts()` now filters by the effective
  flags, so a gated contract is hidden until its flag is set. `regionCleared()`
  was decoupled to the base (ungated) contracts, so the derived
  `home_reconnected` flag (which gates the spoke reveals) stays true after a
  gated contract opens new work. Progress counts (`totalContracts` in journal and
  run-summary) now count in-play contracts, so the board never shows "one you
  cannot see". New e2e field `regionCleared`; hook `version` 7 -> 8.
- Content: one arc-gated contract per region, each appearing after that region's
  reveal and carrying the hidden network's unsigned letters:
  Greybridge `greybridge-follow-the-letters` (requires `greybridge_reveal`, to
  Eastwatch), Saltreach `saltreach-run-the-birds` (requires `saltreach_method`,
  to Reedford), Fenmarch `fenmarch-lamp-for-duskmere` (requires `fenmarch_cost`,
  to Duskmere). All deliver to existing settlements, so world-state status is
  unchanged.
- No save-format change: gating reads persisted `storyFlags`; availability is
  derived, nothing new to persist.
- Tests: `contract-system.test.ts` covers the gating and in-play logic; the two
  region contract-count tests now assert three standing plus one gated;
  `tests/e2e/gated-contract.spec.ts` boots an end-of-Greybridge save and asserts
  the gated contract appears, the delivered ones do not, and `regionCleared`
  stays true (the decoupling).
