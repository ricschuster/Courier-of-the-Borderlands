## Completed

Fog resize migration (PR #31):
- Fog is saved per region as row-major revealed tile indices, which only mean
  the same tile on a same-sized map. The save now also records each region's
  map `[width, height]` in a new optional `fogDimsByRegion` field.
- `fogDimsMatch(stored, width, height)` in `fog-of-war.ts` is pure and unit
  tested. `restoreFog()` in `map-scene.ts` discards a region's fog when the
  stored size does not match the current map (or is absent, as in pre-dimension
  saves); `save()` re-records the current size.
- `SAVE_VERSION` is unchanged, so existing saves keep coins, reputation,
  unlocks, and upgrades; only stale fog is dropped. Pre-dimension saves lose all
  fog once (a one-time re-exploration).

Enlarge Greybridge (PR #32):
- Map grown 20x11 -> 30x22 (about three viewport-screens; camera follow, already
  in place from the prior session, kicks in).
- River splits the region top to bottom with three crossings: an open northern
  bridge, the main central bridge, and the southern ford that still starts
  locked as the unlockable shortcut. The road network forms loops, so there is
  real route choice.
- Two new terrain types: `hills` (northern moor, 0.75x speed) and `marsh`
  (south-east reeds, 0.45x, the slowest passable terrain).
- Content grew from 4 settlements / 3 contracts to 6 / 5:
  - Northcairn: hill-clan settlement on the northern road (letters contract).
  - Mirewatch: stilt-village in the south-east marsh (secrets contract; a good
    payoff for opening the ford).
  - Existing settlements repositioned; spawn, gateways (Saltreach at the east
    edge x=29, Fenmarch on the Southmill tile), and the ford signpost updated.
- Verified in a browser (fresh game): fog present, camera follows, all five
  contracts appear on the Greywater board, and the ford unlocks on the new map.

  `src/systems/region-system.ts`.
- New/changed this session: `fogDimsByRegion` on the save snapshot and
  `fogDimsMatch` in `fog-of-war.ts`; `hills` and `marsh` in `terrain-types.ts`;
  the 30x22 `greybridge-map.ts` and its content modules.

Map authoring reference (learned this session, worth reusing): a region resize
touches the map rows + legend, `settlements-*.ts`, `contracts-*.ts`, and
`region-system.ts` (spawn/gateways/signpost); new terrain symbols need entries
in both the legend and `terrain-types.ts`. Greybridge-specific unit tests that
hard-code specifics: `tile-map.test.ts` (dims + crossing coords) and
`greybridge-content.test.ts` (settlement roster). Most e2e reads coords from the
`?e2e` state hook and needs no change; `ford-route.spec.ts` was the exception
and now derives its staging tiles from the signpost.
