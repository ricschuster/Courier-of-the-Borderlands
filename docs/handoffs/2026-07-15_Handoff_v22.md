## Decisions

- **Telemetry schema bumped to 2, v1 records dropped on parse.** Defaulting old
  records to `'play'` would have mislabelled the hand-seeded bot arc as human
  play, which is the exact confusion the field exists to prevent. The only v1
  history was that seed. Note the untracked `telemetry-seed.*` files in the repo
  root are v1, so the dashboard now ignores them.
- **actionlint is blocking, pinned to a checksummed release binary.** The
  failure mode it guards is a silently broken `main` (#246), so advisory was not
  enough. Pinned so a new actionlint release cannot turn `main` red without a
  commit. Costs about 4s.
- **`security.yml` is not a required check.** `check` and `e2e` remain the only
  gates. The weekly schedule is the point: a dependency clean today can be
  vulnerable next month with no commit touching the repo. The audit fails on a
  high or critical advisory in a runtime dependency (there is one, phaser) and
  reports dev-dependency advisories without failing, since those never ship.
- **Coverage thresholds are a ratchet, not an aspiration.** Set just under the
  real numbers (95.9 stmts, 91.97 branches, 99.3 funcs, 95.94 lines across all
  39 systems). Scoped to `src/systems` only: gating Phaser-bound scenes would
  push toward testing the renderer instead of the rules. Coverage was already
  strong, so this locks in existing practice rather than chasing a number.
- **A dated reminder issue was replaced by a condition check.** #243 was a
  calendar wearing an issue's clothes: a todo nobody could act on for five
  months, sitting in `gh issue list` against the repo's `open = todo`
  convention. `token-expiry.yml` reads the real expiry from GitHub, catches
  revocation and scope loss as well as expiry, stays silent while there is
  runway, and files its own labelled issue inside 45 days. It runs on PRs that
  touch its own file, so it self-tests when edited.
