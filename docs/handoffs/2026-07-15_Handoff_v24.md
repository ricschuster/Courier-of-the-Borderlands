## Three verification lessons worth keeping

These cost real time and will recur:

1. **A green suite proved nothing twice today.** #274 was a function with no
   caller, which no unit test could catch. #278's production half stays green
   even if the base read is completely broken, because "always returns the
   production base" and "correct" are indistinguishable there. Both needed the
   negative case driven deliberately: neutralize the fix, watch the test fail,
   restore.
2. **#227's first attempt was invisible while every test passed.** 3px particles
   against a 48px tile are smaller than the speckles in the Kenney grass texture,
   so they read as terrain. Only a screenshot of real play caught it. Juice is
   visual; tests cannot see it.
3. **The e2e suite structurally cannot test a preview build**, since it only ever
   builds with the production base. #278's real proof needed a hand-built
   `BASE_PATH` deploy. Note `vite preview` reads `base` from the same env var, so
   the server must be given what the build was or nothing runs.

## Risks or decisions needed

- **`googleapis/release-please-action@v4` targets Node 20**, which GitHub has
  deprecated and is force-running on Node 24. It works today. Deliberately not
  filed as an issue: nothing is actionable until upstream ships a release, and an
  issue nobody can act on is a calendar wearing an issue's clothes (the v22
  reasoning that retired #243). Watch it on release runs. Our own actions are
  current (`checkout@v7`, `setup-node@v6`).
- **Any new client-side storage must route its key through `namespacedKey`**
  (`src/systems/storage-namespace.ts`). Path is not an isolation boundary in the
  browser. Recorded in ADR 0008's consequences.
- **The coverage gate is tighter than it looks.** Before #221's storage tests it
  sat at 95.01 statements and exactly 95.00 lines against a 95 threshold: one
  uncovered line from red. A new `src/systems` file whose storage wrapper is
  untested will trip it. The `save-storage` / `error-log-storage` jsdom split is
  the pattern to copy.
- **The untracked `telemetry-seed.*` files** are still in the repo root, still v1
  schema, still ignored by the dashboard. Harmless; delete when convenient.
- **The tractable backlog is empty again**, for the second time today. The queue
  emptied because the verified work ran out, not because game work was
  deprioritised. The next direction is an owner call.
