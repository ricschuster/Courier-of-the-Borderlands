# Session Handoff: 2026-07-10

## Summary

Tackled the player-progression flatline. Introduced a capability-key traversal
mechanic ("roads gate the wagon") and shipped the first two slices: the pure
capability core, and the first gated content (a premium contract behind a
wagon-gated mire). Also built and hardened an autonomous full-arc playthrough
harness and reworked CI so the heavy playthrough runs on-merge and nightly
rather than on every PR.

## Completed

  owner-directed direction (roads gate the wagon; skills as alternative keys;

  never the *only* route. Sliced 1-5.
- **Slice 1 (shipped): capability-key passability core.**
  - `src/systems/traversal.ts` (new): pure `traversalKeys(unlocks, upgrades,
    skills)` returning the set of capability tokens the player holds, plus a
    data-driven `CAPABILITY_GRANTS` map (`mire-crossing` <- `marsh-treads`).
  - `src/scenes/map-scene.ts`: passability now uses `traversalKeys(...)`
    computed once per pathfind (hoisted, not per-tile), across the ~4 pathfinding
    / collider call sites. E2E hook `version: 11` with a new `pickup` field and a
    `?turbo` speed factor (2x, test-only).
  - Unit tests for the pure logic.
- **Slice 2 (in flight, PR #97): premium contract behind a mire.** Reedgrave
  marsh pocket in Greybridge: a water channel at column 26 (rows 16-21) with a
  single gated `deep-mire` crossing at (26,19), new settlement Reedgrave at
  (28,19), and premium contract "The Drowned Cache" (Mirewatch -> Reedgrave,
  reward 120, minRep 8). Marsh Treads opens the crossing; the long way round
  still works. Unit tests assert the crossing is blocked base / open with
  `mire-crossing`, and that the delivery is reachable base but shorter with the
  key.
- **Autonomous playthrough harness.**
  - `tests/e2e/full-arc.spec.ts`: boots the real built game and greedily plays
    the whole three-region arc with real key presses to the blockade-broken
    capstone. Self-correcting postmaster talk (re-seat on home + retry press +
    mark-done-only-on-success). Turbo halves wall-clock (~3m locally).
  - `scripts/autoplay.mjs`: the screenshotting dev-tool sibling of the arc spec.
  - `tests/e2e/drive.ts`: `bootE2E(page, { turbo })` appends `&turbo=1`.
- **CI rework (`.github/workflows/ci.yml`).** `check` + `e2e` run on every PR and
  push (fast). The heavy `arc` job (`RUN_ARC=1`, full playthrough) runs only on
  push-to-main and the nightly `0 7 * * *` cron, never on PRs. The arc spec
  skips itself in CI unless `RUN_ARC` is set. (Note: an earlier separate
  `arc.yml` was folded back into `ci.yml`; there is no separate arc workflow now.)

## Current state

  passes locally (~3.2m) and delivers the new 6th standing Greybridge contract.
  The capability mechanic is live and the ford still works through it.
- **In flight:** PR #97 (`feat/reedgrave-mire-pocket`) has auto-merge armed
  (squash) and its branch was just updated to catch up to main. It should land
  on its own once `check` + `e2e` pass. Verify it merged.
- **Known issues / caveats:**
  - The `arc` job runs *after* merge, so an arc-breaking change is caught on main,
    not pre-merge. Open decision (deferred): make `arc` a required check on
    branch protection to block merges, accepting the ~3m per-PR cost, OR add a
    pre-merge arc gate only on `src/**`-touching PRs. Not yet done.
  - The arc driver does not buy upgrades or rank skills, so gated content must
    always leave the critical path open or the arc soft-locks. Slice 4 addresses
    this.
